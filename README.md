# AI Image Detection Bypass

对抗 AI 生成图片检测工具的脚本。通过 FFT 频谱匹配、相机管线模拟、噪声注入、纹理归一化等方法，修改图片的统计特征以绕过 AI 图像检测器。

## 检测器怎么判断一张图是不是 AI 生成的

检测器 (ai_forensics_lab.py) 做三件事：

1. 将图片高斯模糊后与原图做差 → **噪声残差 (noise residual)**
2. 对原图做 FFT → **频谱 (spectrum)**
3. 频谱按频率径向分 32 个桶 → 计算 **high/low 频率比**

AI 图的特点：**residual_std 偏低（噪声太少/太规整），high/low 频率比偏低（高频能量不足）。**

```
输入图 → 灰度化
         ├─→ FFT → 频谱幅度 → 径向分32个bin
         │         → low = bin[1:8]均值
         │         → high = bin[-8:]均值
         │         → high/low 比值  ← AI图这个值偏低
         │
         └─→ 高斯模糊(radius=3) → 原图-模糊图 = residual
                   → residual_std    ← AI图这个值偏低
                   → residual_abs_mean
```

真实相机拍摄的图 high/low 约 0.12~0.15，AI 图约 0.02~0.05。本工具的目标就是把这两个值拉到接近真实相机的水平。

---

## 核心思路

AI 生成图太「干净」了。每一步都在不同维度上制造「不完美」，让这些不完美符合真实相机的物理特性。

## 7 步管线详解

### 第 1 步：DFL 频谱对抗

**目的**：用梯度下降在像素空间找最微小的扰动，最大化 FFT 频谱变化

**原理**：DFL = Differential Frequency Loss。把图片当作一个可微的函数，用梯度下降（类比「蒙眼下山」——每次摸一下脚边哪个方向往下，就往那走一小步）搜索一组像素扰动 δ，让 FFT 频谱变化最大，同时用 LPIPS（感知损失）和 L2（扰动幅度）两个约束保证人眼完全看不出变化。

```
下山类比：
  你站在山上蒙着眼 → 目标是走到山谷最低点
  每步摸一下 → 哪个方向往下倾斜 → 走一小步 → 重复

DFL 也是：
  当前图 → 算梯度（每个像素调1个单位，频谱会变多少？）
  → 按梯度方向调像素 → 重复80轮 → 频谱被搅得面目全非
  但 LPIPS + L2 约束保证画面不变
```

**为什么是第一步**：DFL 先「破防」（打乱原频谱结构），FFT 再「重建」（按 1/f 曲线重建），顺序不能反。反过来 FFT 先掰好频谱，DFL 会把 FFT 的努力重新搅乱。

### 第 2 步：FFT 频谱匹配

**目的**：将频谱掰向 1/f^α 自然幂律分布

**频率是什么**：
```
图像的频率 = 像素变化的快慢

低频区域（变化慢）：天空、墙壁、大块均匀颜色
高频区域（变化快）：树叶边缘、头发丝、噪点、锐利的边

傅里叶变换做的事：
  「这张图里，低频有多少？中频有多少？高频有多少？」
  像调音师听一段音乐能说出「这个和弦里有多少低音、多少中音、多少高音」
```

**FFT 输出两个东西**：

- **幅度 (magnitude)**：每个频率的「音量」有多大。像均衡器 (EQ) 上的推子——低音柱多高、高音柱多高
- **相位 (phase)**：每个频率的条纹「对齐到哪」。相位承载了图像的结构信息——只改幅度不改相位，**图像内容不会变，只有纹理质感会变**

**1/f 幂律**：自然图像的频谱幅度 ≈ 1/频率。频率翻倍，能量减半。AI 生成的图偏离这个曲线——中高频衰减得太快。

**具体操作**：
```
① 把图转 LAB，只取 L（亮度）通道
② 做 FFT，拆成幅度 + 相位
③ 构建目标幅度 = (1/f)^(α/2)，α=1.0
④ 低频（r < 0.15）保护不动（低频=天空/皮肤，动了会变色）
⑤ 中高频替换成目标幅度
⑥ 新幅度 + 原相位 → 逆 FFT → 回到图像
⑦ 和原图混合：65% 新 + 35% 旧（strength=0.65）
⑧ 加微小随机性打破规律
```

**为什么只改亮度不改颜色**：检测器的 FFT 分析是在灰度上做的，改颜色没用。在 LAB 空间里 L 通道是纯亮度，改了不会导致颜色失真。

**为什么这是最重要的一步**：检测器的核心指标 high/low 频率比，FFT 匹配直接对着这个指标的命门打——人为提升中高频能量。你实验证实这步让 high/low 跳了 3.52。

### 第 3 步：相机管线模拟

**目的**：模拟真实相机从光子到 JPEG 的完整物理过程

**真实相机的工作流程**：
```
场景光 → 镜头(暗角+色差) → Bayer CFA(每个像素只记录一种颜色)
  → 传感器(光子→电子，Poisson散粒噪声)
  → 读出电路(Gaussian热噪声)
  → 去马赛克(把缺失的颜色用邻居插值猜回来)
  → 白平衡 → 色彩矩阵 → gamma校正(线性→sRGB)
  → JPEG压缩
```

**每一步的作用**：

- **Bayer 马赛克化**：每个像素只剩一种颜色（RGGB 排列），故意扔掉 2/3 的颜色信息
- **去马赛克**：用邻居插值把颜色猜回来。猜的过程会产生微弱的插值伪影——相邻像素之间有去马赛克算法留下的相关性，这是真实相机的重要指纹，AI 图没有
- **Poisson-Gaussian 噪声**：光子散粒噪声（光越强噪声绝对量越大） + 读出电路热噪声（恒定的噪声底）
- **暗角 (Vignetting)**：边缘比中心暗，cos⁴ 衰减
- **色差 (Chromatic Aberration)**：不同波长光折射角度不同 → 物体边缘微弱的红/蓝镶边
- **JPEG 压缩**：有损压缩的 8×8 分块伪影，多次压缩累积

**当前实现的顺序**：
```
Bayer → 去马赛克 → 色差 → 暗角 → 传感器噪声 → 坏点 → JPEG
```

**已知问题**：噪声加在了 sRGB 域（gamma 校正后）而非线性 RAW 域。真实相机的噪声发生在传感器线性阶段，然后才经过 gamma 校正。sRGB 的暗部被 gamma 拉伸、亮部被压缩，在 sRGB 域加噪会导致噪声分布不自然。正确顺序应该是：逆 gamma → Bayer → 噪声 → 去马赛克 → 白平衡 → gamma → 暗角+色差 → JPEG。

### 第 4 步：GLCM 纹理归一化

**目的**：改变纹理统计分布，让 AI 图的 homogeneous 纹理更接近真实相机

**GLCM (灰度共生矩阵) 是什么**：
问一个问题：「如果一个像素的亮度是 X，那它右边邻居的亮度是 Y 的概率是多少？」GLCM 就是一张统计表，记录了所有亮度值的邻居关系。

从 GLCM 可以提取两个关键特征：
- **Contrast（对比度）**：邻居之间亮度差大不大。差大 → 纹理丰富
- **Homogeneity（均匀度）**：邻居之间亮度有多相似。相似 → 纹理单调

**AI 图的问题**：homogeneity 太高、contrast 太低。像素和邻居太像了。

**操作**：
```
① 转 LAB 色彩空间，只改 L（亮度）通道
② 计算当前 GLCM → 得到 contrast 和 homogeneity
③ 设定目标：contrast 提升 ×1.3，homogeneity 降低 ×0.85
④ 根据比例缩放亮度通道 → 相邻像素对比度拉大
⑤ 双边滤波保边 → 纹理改了但边缘轮廓不糊
⑥ 加微噪声 → 模拟真实纹理的随机性
```

### 第 5 步：LBP 纹理归一化

**目的**：增加局部二值模式的多样性，让纹理模式更丰富

**LBP (局部二值模式) 是什么**：
对于每个像素，比较它和周围一圈邻居的亮度。邻居比中心亮 → 1，比中心暗 → 0。按固定顺序读出，得到一个二进制数。这就是这个像素的 LBP 值，它描述了「这个像素周围的明暗分布模式」。

```
一个像素的 3×3 邻居：
  120<125→0   130>125→1   110<125→0
  115<125→0   中心125     135>125→1
  140>125→1   118<125→0   128>125→1

按顺序读出 → 01001101 → LBP 值
```

**LBP 直方图**：统计全图所有像素的 LBP 值分布。
- AI 图：直方图集中在少数几个值上。纹理模式很单调，到处都是类似的局部模式
- 真实相机：分布更均匀。传感器噪声、Bayer 插值、JPEG 压缩在微观层面创造了丰富多样的纹理

类比：AI 图像用 3 种乐高积木搭的，真实相机用了 50 种。看起来差不多，显微镜下完全不同。

**操作**：
```
① 灰度化 → 计算每个像素的 LBP 值
② 统计直方图 → 计算 CDF（累积分布函数）
③ 目标 CDF = 当前 CDF 和均匀分布的混合
   → 把分布拉平，让罕见模式出现，常见模式抑制
④ 根据目标 CDF 为每个像素算缩放系数 → 调亮度
⑤ 加微噪声
```

你实验证实 LBP 关掉后 high/low 从 3.99 降到 2.45，说明 LBP 对频域有显著影响（纹理模式的变化反映在中高频）。LBP 对 residual 贡献约 +1.15。

### 第 6 步：高斯噪声

**目的**：直接在像素空间叠加符合正态分布的随机波动，拉高 residual_std

**为什么是高斯分布**：中心极限定理——大量独立小因素叠加自然趋近高斯分布。真实传感器电路热噪声就是大量电子随机运动叠加的结果。

```python
std = noise_std_frac × 255  # 当前 0.012 × 255 ≈ 3.06
# 每个像素加上均值 0、标准差约 3 的随机数（像素范围 0-255）
# 68% 的值在 ±3 内，95% 在 ±6 内
```

**和第 7 步的区别**：

| | 高斯噪声（第6步） | 像素扰动（第7步） |
|---|---|---|
| 分布 | 正态分布（钟形，集中在0附近） | 均匀分布（范围内每个值概率相等） |
| 强度 | 68%在 ±3 内，极端可达 ±9 | 严格在 ±2 内 |
| 目的 | 模拟传感器物理噪声 | 打破像素间残留的规律性 |
| 类比 | 砂纸打磨，有深有浅 | 橡皮锤均匀敲一遍 |

### 第 7 步：像素扰动

**目的**：用微弱但无规律的均匀偏移打乱任何残留的像素间相关性

AI 生成图有一种非常细微的规律性——扩散模型去噪后残留的网格状伪影 (grid artifact)、GAN 上采样留下的棋盘格 (checkerboard artifact)。这些在人眼看来不明显，但像素级检查时会被发现。均匀扰动每个像素独立加随机偏移，原本可能有的「这几个像素的偏差方向一致」的模式被打破。

每个 R、G、B 通道独立加扰动，色相也会微调，更接近真实传感器每个像素独立曝光的特性。

### 第 8 步：EXIF 伪造

**目的**：写入真实相机品牌/型号/镜头/拍摄参数的假 EXIF 元数据

**为什么需要**：很多 AI 生成图完全没有 EXIF，或只有软件信息（"Created with Midjourney"）。检测器检查 EXIF 是一个很便宜的判断——没有相机信息 → 高度可疑。

**操作**：从 10 个品牌-型号-卡口严格配对的组合中随机选择，生成合理的拍摄参数（焦距 24-135mm、光圈 1.4-11、ISO 100-1600、快门 1/60-1/2000），嵌入标准 EXIF 字段。

**为什么品牌和卡口要配对**：如果 EXIF 里 Make="Sony" 但 LensModel 写的是 "XF 50mm f/2.8"（富士的卡口），稍微懂行的人一眼就能看出是假的。组合严格配对：Canon EF 相机 → EF 卡口镜头、富士 → XF 卡口、徕卡 → Summilux。

## 为什么 AI 生成图缺少高频

**扩散模型的本质是「去噪」，天然杀高频**：每一步都在消除噪声，而噪声就是高频。模型被训练成「把高频去掉」的专家，宁可少留高频也不愿把噪声残留下来（残留噪声比「不够锐」明显得多）。

**VAE 的 8 倍压缩**：Stable Diffusion 不是在像素空间工作的。它先把图压缩 8 倍（VAE encoder），在 64×64 的潜空间做扩散，再放大回来。高频信息在压缩时就丢了大半，decoder 脑补回来的纹理不够自然。

**损失函数的「平均化」效应**：MSE（均方误差）倾向于输出所有可能答案的「平均值」，而不是任意一个「锐利」的答案。这导致生成的图偏向光滑。

**这不是能力问题，是设计取舍**：AI 公司追求画面干净好看，用户觉得噪点 = 画质差。他们服务的是觉得噪点 = 烂图的用户，不是检测器。

## 已知问题

- high/low 频率比仍偏高（3.20 vs 目标 0.13，内部指标，与检测器指标不同量纲）
- residual_std 偏低
- 相机管线噪声在 sRGB 域而非线性 Bayer 域，顺序不正确
